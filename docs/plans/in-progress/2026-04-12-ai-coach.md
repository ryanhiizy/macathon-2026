# AI Coach Feature Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a daily AI-generated coaching insight that appears on the Habits tab. The coach analyzes a user's habit history — completion patterns, streaks, timing, group vs solo — and delivers one specific, actionable insight per habit per day.

**Why it matters for the demo:** Judges see it immediately on the main screen without navigating anywhere. It proves the app learns from user behavior and delivers personalized value — not just tracking, but coaching.

**Architecture:** The frontend requests an insight from the FastAPI server. The server queries Supabase for the user's habit stats, computes a stats summary, sends it to Claude, and returns a structured insight. For the hackathon MVP, insights are generated on-demand (no caching table needed). If the user has insufficient data, the server returns a hardcoded "starter" insight.

**Tech Stack:** FastAPI endpoint, Anthropic Claude API (already configured), Supabase PostgREST queries from Python, React Native card component.

## Task 1: Backend — Stats computation module

**Files:**
- Create: `backend/services/coach.py`

**What to build:**

A `compute_habit_stats()` function that takes a Supabase client + user_id + habit_id and returns a stats dict. This is the analytics engine that powers everything.

**Stats to compute:**

```python
@dataclass
class HabitStats:
    habit_name: str
    category: str
    total_instances: int          # all habit_instances for this habit
    verified_count: int           # instances with status='verified'
    missed_count: int             # instances with status='missed'
    completion_rate: float        # verified / total (last 30 days)
    current_streak: int           # from circle_members
    best_streak: int              # from circle_members
    avg_completion_hour: float    # mean hour-of-day when snaps were created
    best_day_of_week: str         # day with highest completion rate
    worst_day_of_week: str        # day with lowest completion rate
    group_prove_count: int        # snaps where is_group_post=true
    solo_prove_count: int         # snaps where is_group_post=false
    days_since_creation: int      # how long the user has had this habit
    has_enough_data: bool         # true if verified_count >= 5
```

**Queries needed (all scoped to user_id + habit_id, last 30 days):**

1. Get habit metadata: `SELECT name, category FROM habits WHERE id = $habit_id AND user_id = $user_id`
2. Get instance counts: `SELECT status, COUNT(*) FROM habit_instances WHERE habit_id = $habit_id GROUP BY status`
3. Get snap timing: `SELECT created_at, is_group_post FROM snaps WHERE user_id = $user_id AND habit_instance_id IN (SELECT id FROM habit_instances WHERE habit_id = $habit_id)`
4. Get streak: `SELECT current_streak, best_streak FROM circle_members WHERE user_id = $user_id AND circle_id = (SELECT circle_id FROM habits WHERE id = $habit_id)`

**How to query Supabase from Python:**
Use the `supabase-py` client or just use `httpx` to hit the PostgREST API directly. For the hackathon, raw HTTP with the service role key is fastest. Add `SUPABASE_URL` and `SUPABASE_SERVICE_KEY` to `backend/.env` and `backend/config.py`.

## Task 2: Backend — Insight generation via Claude API

**Files:**
- Modify: `backend/services/coach.py` (add `generate_insight()`)

**What to build:**

A `generate_insight(stats: HabitStats) -> CoachInsight` function that:

1. Checks `stats.has_enough_data` — if false, return a hardcoded starter insight
2. Builds a stats summary string from the HabitStats
3. Calls Claude with a system prompt + the stats summary
4. Parses the response into a structured `CoachInsight`

**System prompt:**

```
You are a habit coach for an app called "presence". You analyze a user's habit data and generate ONE short, specific, encouraging insight.

Rules:
- Output exactly two lines: HEADLINE and DETAIL
- HEADLINE: one punchy sentence, max 12 words, action-oriented or celebratory
- DETAIL: one sentence with a specific number/stat backing up the headline
- Reference the habit by name
- Be warm and encouraging, never preachy or guilt-trippy
- If there's an actionable pattern (best time, best day, group effect), highlight it
- Pick the most interesting/useful pattern, not the most obvious one
```

**Insight type classification** (done post-generation based on which stats were most notable):

| Type | Trigger condition |
|---|---|
| `optimal_timing` | Clear best hour (>70% of completions in a 2-hour window) |
| `streak_momentum` | Current streak >= 7 and is the longest or near-longest |
| `risk_alert` | One day of week has <30% completion rate |
| `social_boost` | Group prove completion rate > solo by 30%+ |
| `cross_habit` | (Future: skip for MVP) |
| `recovery` | User broke a streak recently but is rebuilding |
| `general` | Default fallback |

**Response model:**

```python
@dataclass
class CoachInsight:
    headline: str       # "You crush gym sessions before 8am"
    detail: str         # "90% completion rate vs 40% after noon"
    habit_name: str     # "Morning gym"
    insight_type: str   # "optimal_timing"
```

**Fallback:** If Claude API fails or no key is configured, return a hardcoded insight based on the stats (e.g., just report the current streak).

## Task 3: Backend — `/coach-insight` endpoint

**Files:**
- Modify: `backend/prompt_verification_server.py`

**What to build:**

Add a new POST endpoint that follows the existing pattern:

```python
class CoachInsightRequest(BaseModel):
    user_id: str
    habit_id: str

class CoachInsightResponse(BaseModel):
    headline: str
    detail: str
    habit_name: str
    insight_type: str

@app.post("/coach-insight", response_model=CoachInsightResponse)
async def get_coach_insight(request: CoachInsightRequest) -> CoachInsightResponse:
    stats = await compute_habit_stats(request.user_id, request.habit_id)
    insight = await generate_insight(stats)
    return CoachInsightResponse(
        headline=insight.headline,
        detail=insight.detail,
        habit_name=insight.habit_name,
        insight_type=insight.insight_type,
    )
```

**Also add:**
- `SUPABASE_URL` and `SUPABASE_SERVICE_KEY` to config.py settings
- `supabase` or `httpx` to requirements.txt
- Update `/health` to report coach readiness (Supabase connection status)

## Task 4: Frontend — CoachInsightCard component

**Files:**
- Create: `frontend/components/CoachInsightCard.tsx`

**What to build:**

A self-contained card component that:
- Takes `habitId` and `userId` as props (or fetches from auth context)
- Calls the `/coach-insight` endpoint on mount
- Renders the insight with this layout:

```
┌─────────────────────────────────────────┐
│  🧠  AI Coach                      ✕   │
│                                         │
│  You crush gym sessions before 8am      │  ← headline (bold, 16px)
│  90% completion rate vs 40% after noon  │  ← detail (muted, 14px)
│                                         │
│  ┌──────────┐                           │
│  │ Morning gym │                        │  ← habit tag pill
│  └──────────┘                           │
└─────────────────────────────────────────┘
```

**Design details:**
- Background: subtle gradient or tinted card (use theme accent at 8% opacity)
- Icon: brain or sparkle icon from HugeIcons
- Dismiss: X button in top-right, sets `dismissed` state (AsyncStorage keyed by date + habit)
- Loading: skeleton shimmer while fetching
- Error/no-data: don't render the card at all (return null)
- Animation: subtle fade-in on mount

**State management:**
- `useState` for insight data, loading, dismissed
- `useEffect` to fetch on mount
- AsyncStorage to persist dismissal per day
